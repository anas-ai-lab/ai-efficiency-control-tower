## Was unerwartet war

**Schema-Konflikt:** `models.py` existierte bereits mit `frozen=False` und
`EvidenceQuality` statt `EvidenceLevel`. Drei Iterationen bis zur finalen
konsistenten Version — ursprüngliche Guide-Anleitung hatte die bereits
bestehenden Dateien nicht berücksichtigt.

**venv-Debugging:** `ModuleNotFoundError: No module named 'aect'` trotz
korrektem `pyproject.toml`. Ursache: macOS hatte beim editable install eine
zweite `.pth`-Datei mit Leerzeichen im Namen angelegt
(`_editable_impl_aect 2.pth`), die einen doppelten Pfad enthielt
(`/Users/.../src/Users/.../src`). Fix: `rm -rf .venv && uv sync`.

**ruff RUF002:** `×` (Multiplikationszeichen, Unicode U+00D7) in einem
Docstring wird als ambiguous character gewertet. VS Code hat das Zeichen
nicht korrekt ersetzt — `sed -i` im Terminal hat es gelöst.

## Architektur-Entscheidung (für Interview)

`frozen=True` ist keine Stilentscheidung. Das Application-Layer darf
`UseCaseInput` nicht mutieren — Berechnungsergebnisse (ROI, Zone, Verdict)
gehören in separate Ausgabeobjekte (`TriageResult`). Würde das Input-Objekt
angereichert, vermischt man Eingabe- und Berechnungsschicht — das bricht
Hexagonal Architecture.
