## Debugging-Session: Hatchling Editable Install

**Symptom:** `uv run python -c "from aect.domain import..."` schlug fehl
mit `ModuleNotFoundError: No module named 'aect'`. `uv run pytest` lief
trotzdem grün — das hat das Problem kaschiert.

**Root Cause:** hatchling's neuer editable-install-Modus erstellt einen
Python Import-Hook (`_editable_impl_aect.pth`) statt einer einfachen
Pfad-Datei. Der Hook zeigt auf das Projekt-Root statt auf `src/`.
`uv sync` / `uv add` baut das Paket neu und überschreibt jeden manuellen
Fix. pytest funktionierte wegen `pythonpath = ["src"]` in der pytest-Config
— ein falsches Sicherheitsgefühl.

**Fix:** Build-Backend auf `setuptools.build_meta` + `[tool.setuptools
.packages.find] where = ["src"]`. setuptools erstellt eine einfache
`.pth`-Datei, die `uv sync` überlebt.

**Lernpunkt:** editable install sofort nach jedem `uv add` testen, nicht
nur `uv run pytest`. Die Kombination pytest-pythonpath + defektes editable
install ist ein stiller Fehler.
