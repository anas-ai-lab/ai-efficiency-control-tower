## Comprehension Gate

**Kontext:** Der Eval-Runner ruft gar kein KI-Modell auf -- er schickt
Use Cases durch die Rechenregeln (ROI, Zonen, Scoring). Ein mock/azure-
Schalter haette nichts umgeschaltet. Praeduktive Validitaet wuerde
voraussetzen, dass Use Cases produktiv umgesetzt werden und reale
Ergebnisse zurueckfliessen -- im privaten Build nicht moeglich.

**Frage 1:** Warum haette ein mock/azure-Schalter im Eval-Runner nichts
gebracht?

**Antwort:** Der Runner ruft kein LLM auf -- nur Rechenregeln. mock und
azure waeren identisch. Claude ergaenzt: ein Schalter der nichts
umschaltet ist eine Leerform, die nach aussen eine Auswahl suggeriert
die innen nicht existiert.

**Frage 2:** Was fehlt fuer echte praediktive Validitaet?

**Antwort:** Feedback aus der Realitaet -- Use Cases muessen produktiv
umgesetzt werden und die tatsaechlichen Ergebnisse muessen zurueck-
fliessen. Im privaten Build gibt es keine solche Vergleichsbasis.
