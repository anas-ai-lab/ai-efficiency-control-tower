## Was heute gebaut wurde

Zwei neue Regel-Module in der Domain-Schicht:

**Vorfilter (`filters.py`):** Prüft drei Mindestkriterien bevor ein Use Case tiefer bewertet wird — theoretisches Potenzial ≥ 20.000 €, Stundeneinsparung ≥ 120 h/Jahr, Nettonutzen ≥ 5.000 €. Ergebnis ist ein `FilterResult`-Dataclass mit `passes: bool`, der Liste der fehlgeschlagenen Kriterien und einem Detail-Dict für den Audit-Trail. Schwellenwerte sind als Default-Parameter ausgelegt — in Phase B kommen sie aus der Config (IP-Trennung).

**Composite-Aufwand-Score (`scoring.py`):** Berechnet einen Aufwands-Score aus drei Dimensionen: technische Komplexität (1–5), Implementierungskosten (1–3), Datenschutzstufe (0–2). Summe ergibt einen Wert zwischen 2 und 10, der automatisch in ein Textlabel übersetzt wird: NIEDRIG / MITTEL / HOCH. Das Mapping von `DataClassification` auf Datenschutz-Punkte spiegelt die DSGVO-Logik: NO_PERSONAL_DATA = 0, PSEUDONYMOUS = 1, PERSONAL und SENSITIVE_PERSONAL = 2 (Art. 9 DSGVO ist nicht "mehr" als Art. 4 — beide verlangen höchste Vorsicht).

## Was schiefgelaufen ist

**Problem 1 — Enum-Werte falsch im Template:** Die Test-Datei hatte `DataClassification.PUBLIC`, `DataClassification.CONFIDENTIAL` usw. — die im Projekt nicht existieren. Die echten Werte lauten `NO_PERSONAL_DATA`, `PSEUDONYMOUS`, `PERSONAL`, `SENSITIVE_PERSONAL`. Ursache: ich hab die Enum-Namen aus Schritt 2 nicht in die Tests übertragen, bevor ich sie eingefügt habe. Fix: Tests auf echte Enum-Werte angepasst.

**Problem 2 — `python` statt `uv run python`:** Der Verifikationsbefehl in der Anleitung fehlte `uv run`. System-Python kennt `aect` nicht, nur das Projekt-Venv. Immer `uv run python`, nie `python` direkt.

**Problem 3 — Build-Backend ignoriert `src/`:** Das Paket war installiert (`uv pip list` zeigte es), aber `import aect` schlug trotzdem fehl. Ursache: das editable install erstellt eine `.pth`-Datei, die auf das Projekt-Root zeigt — Python sucht dann nach `aect/` im Root, aber es liegt in `src/aect/`. Fix: `pyproject.toml` braucht `packages = ["src/aect"]` unter `[tool.hatch.build.targets.wheel]` damit der Build-Backend den richtigen Pfad kennt.

**Problem 4 — RUF001 (wieder):** En-Dashes (`–`) in Error-Strings statt Hyphens (`-`). Gleicher Fehler wie in Tag 16 (roi.py). Fix: `sed`-Befehl hat alle 5 Vorkommen in `scoring.py` ersetzt.
