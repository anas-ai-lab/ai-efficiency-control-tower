# Day 27 — SQLite-Persistenz

**Datum:** 2026-06-11
**Phase:** B (API-Layer & Persistenz)

## Ziel
SQLiteRepository implementiert RepositoryPort. Settings + Dependency-Provider
waehlen Backend per AECT_DB_PATH (leer = InMemory, gesetzt = SQLite).

## Was gebaut wurde

- `src/aect/adapters/sqlite/repository.py`
  - SQLiteRepository (save/get/list_all) -- strukturelle RepositoryPort-Implementierung
  - Serialisierung: UseCaseInput via model_dump_json()/model_validate_json()
    (Pydantic), TriageResult via dataclasses.asdict() + _DecimalEncoder
  - Explizite Deserialisierungsfunktionen pro Domain-Typ (FilterResult,
    RoutingResult, FeasibilityResult, ROIResult, CompositeScore, ZoneResult)
    -- noetig wegen StrEnum-Rekonstruktion und tuple-vs-list nach JSON-Roundtrip
- `src/aect/adapters/sqlite/__init__.py` -- Package-Export
- `tests/adapters/sqlite/test_sqlite_repository.py` -- 16 Tests:
  - Init (4): DB-Datei erstellt, idempotentes CREATE TABLE, leere Liste,
    unbekannte ID -> None
  - CRUD (3): save/get, INSERT OR REPLACE bei Duplikat, list_all sortiert
  - Roundtrip (9): Decimal-Praezision, StrEnum-Rekonstruktion (TriageZone,
    RoutingRecommendation, FeasibilityFlag), tuple-Felder, None-Felder bei
    Vorfilter-Fail
- `src/aect/adapters/api/settings.py` -- `db_path: str = ""` ergaenzt
- `src/aect/adapters/api/dependencies.py`:
  - Import SQLiteRepository, Path, RepositoryPort
  - `get_triage_service()` bekommt `settings: Settings = Depends(get_settings)`
  - `repo: RepositoryPort = SQLiteRepository(...) if settings.db_path else _repository`
    (Ternary statt if/else -- ruff SIM108; explizite RepositoryPort-Annotation
    loest mypy-Typkonflikt zwischen InMemoryRepository und SQLiteRepository)

## Was bewusst nicht angefasst wurde
- Idempotency-Keys (Tag 28)
- Kein Lifespan-Singleton fuer SQLiteRepository -- pro Request neu erzeugt,
  `_init_db()` (CREATE TABLE IF NOT EXISTS) laeuft mit. Idempotent, fuer
  Portfolio-Traffic akzeptabel. Doku-Punkt fuer Phase-B-ADR.
- `_repository`-InMemory-Singleton bleibt als Fallback bestehen (AECT_DB_PATH leer)
