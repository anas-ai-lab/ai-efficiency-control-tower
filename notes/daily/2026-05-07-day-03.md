## Gelernt

- `git init` auf bestehendem Repo ist harmlos.
- GitHub akzeptiert kein Passwort mehr über HTTPS → Personal Access Token nötig.
- `git remote remove` / `git remote add` für Fehlerkorrektur.
- `.gitignore` muss vor public stellen sauber sein.
- README ist das erste was Recruiter und Tech Leads sehen – falscher Fokus schadet dem Portfolio.

## Fehler
