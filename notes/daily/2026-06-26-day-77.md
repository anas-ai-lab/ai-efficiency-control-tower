der Test signalisiert den Drift. Solange jemand bewusst beides anpasst,
ist der Schutz noch da.

Das Gate-Kriterium aus Master-Plan v3.1 Phase F ("System aus frischem Clone
in ~10 Min startbar") wurde noch nicht abgehakt. Vorgehen:

1. Neuen Ordner ausserhalb des Projektordners erstellen
2. `git clone <repo-url> aect-fresh` -- frischer Clone, kein bestehendes .venv
3. README-Schritte linear durchgehen (Quick-Start-Sektion), nicht aus dem Kopf
4. Backend starten (Docker fuer ChromaDB, FastAPI via uv run)
5. Frontend starten (npm install, npm run dev)
6. `uv run pytest -q` im frischen Clone -- alle gruen
7. `curl http://localhost:8000/health` antwortet
8. Zeit stoppen -- Ziel: unter 10 Minuten

Wenn README-Schritte fehlschlagen oder fehlen: sofort dokumentieren und fixen.
Das ist der Unterschied zwischen "laeuft auf meiner Maschine" und "verteidigbar".
