# Day 10 — pytest Config, Smoke Tests, Coverage Setup

**Datum:** 2026-05-08
**Phase:** Woche 2 — Engineering Foundation
**Fokus:** Test-Infrastruktur

## Was gemacht wurde

### 1. Dev-Dependencies installiert
`pytest`, `pytest-cov` und `coverage` via `uv add --dev` hinzugefügt.
`uv.lock` wurde automatisch aktualisiert und separat committed.

### 2. `src/aect/__init__.py` befüllt
`__version__: str = "0.1.0"` und `__author__: str = "anas-ai-lab"` eingetragen.
Diskussion: Die explizite `str`-Annotation ist redundant (mypy inferiert sie),
schadet aber nicht. `__author__` ist Konvention ohne technischen Effekt.

### 3. pyproject.toml bereinigt
Problem: Beim Einfügen der neuen pytest- und coverage-Sektionen entstanden
doppelte TOML-Keys (`[tool.pytest.ini_options]`, `[tool.coverage.run]`,
`[tool.coverage.report]`). TOML erlaubt keine Duplikate — Fehler behoben
durch Entfernen der älteren, kürzeren Blöcke. Nur die vollständigen
neuen Sektionen blieben erhalten.

Konfiguriert:
- `[tool.pytest.ini_options]`: testpaths, addopts (--strict-markers, --tb=short,
  --cov, --cov-report), pythonpath
- `[tool.coverage.run]`: source, branch=true, omit
- `[tool.coverage.report]`: show_missing, fail_under=0, exclude_lines

### 4. `tests/conftest.py` erstellt
Marker-Registrierung via `pytest_configure`: unit, integration, e2e.
Verhindert `PytestUnknownMarkWarning` bei `--strict-markers`.

### 5. `tests/test_smoke.py` erstellt
3 Tests:
- `test_package_importable`: Package lässt sich importieren
- `test_version_is_string`: `__version__` ist ein nicht-leerer String
- `test_version_format`: `__version__` folgt semver (MAJOR.MINOR.PATCH)

### 6. `.gitignore` erweitert
`htmlcov/` und `.coverage` hinzugefügt — Coverage-Artefakte nicht committen.

### 7. Fehler unterwegs
- `.gitignore`: Terminal-Befehle versehentlich in die Datei geschrieben
  statt im Terminal ausgeführt → manuell korrigiert.
- `pyproject.toml`: Doppelte TOML-Sektionen durch zweimaliges Einfügen
  → manuell bereinigt.

## Ergebnis
`uv run pytest -v` → 3 passed, 100% Coverage, htmlcov/ generiert.
