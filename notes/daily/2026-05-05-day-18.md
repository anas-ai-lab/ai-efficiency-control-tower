## Was heute gebaut wurde

Drei zusammenhängende Module, die aus den ROI-Zahlen ein Urteil ableiten:

**`zones.py` — ZoneClassifier:**
Nimmt `expected_benefit_eur`, `composite_score` und `handlungsdruck_score`
und gibt eine `ZoneResult` zurück: Basis-Zone, Finale Zone, ob Handlungsdruck
hochgestuft hat, und einen lesbaren Grund-String.

Drei Basis-Zonen (analog v5-Modell):
- LIKELY_WIN: hoher Nutzen, überschaubarer Aufwand
- CALCULATED_RISK: vertretbar mit Einschränkungen
- MARGINAL_GAIN: zu wenig Nutzen oder zu hoher Aufwand

Handlungsdruck-Elevation: Score ≥ Schwellenwert → Zone um eine Stufe hoch.
Thresholds kommen aus `config/zone_thresholds.yaml` (nicht hartkodiert).

**`feasibility.py` — FeasibilityChecker:**
Strukturelle Qualitätsprüfung — unabhängig vom Geschäftswert. Vier Checks:
1. Ist- und Soll-Zustand ≥ 50 Zeichen (konkret beschrieben)
2. Beispielvorgang ≥ 30 Zeichen (vorhanden)
3. Zeitersparnis pro Vorgang > 0
4. Vorgangshäufigkeit pro Monat > 0

Gibt `FeasibilityResult` zurück: `is_feasible`, `flags`-Tuple, optionale
Deutsche Empfehlung für den Einreicher.

**`config/zone_thresholds.yaml`:**
Schwellenwerte ausgelagert — IP-Trennung nach interne Referenz (entfernt) §5.

## Bugs und Fixes heute

**Bug 1 — Editable Install bricht nach `uv add`:**
`ModuleNotFoundError: No module named 'aect'` bei direktem `python -c`.
Fix: `uv pip install -e . --reinstall`. Tritt nach jedem `uv add` auf.
`uv run pytest` zeigt den Bug nicht (hat `pythonpath = ["src"]` als Fallback).

**Bug 2 — Hypothesis HealthCheck: function_scoped_fixture:**
Property-Tests mit pytest-Fixture schlagen fehl weil Hypothesis das
Fixture nicht zwischen generierten Inputs resettet. `ZoneClassifier` hat
keinen mutablen State → `suppress_health_check=[HealthCheck.function_scoped_fixture]`
ist korrekt und sicher.

**Bug 3 — RUF022: `__all__` not sorted:**
ruff erzwingt alphabetische Sortierung. Fix:
`uv run ruff check --unsafe-fixes --fix src/aect/domain/__init__.py`
