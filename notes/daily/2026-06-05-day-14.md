# Day 14 — 2026-06-05

## Was war das Ziel?

Erster Phase-A-Tag: Domain-Layer-Struktur anlegen, erstes Pydantic V2
Input-Schema (UseCaseInput) mit TDD entwickeln.

## Was wurde gebaut?

### `src/aect/py.typed`
Leere Marker-Datei (PEP 561). Signalisiert mypy dass src/aect/ vollständig
typisiert ist. Ohne sie kann mypy Importe aus dem Package in manchen Kontexten
als Any behandeln — dann ist mypy strict wirkungslos.

### `src/aect/domain/models.py`
Pydantic V2 Input-Schema mit zwei Klassen:

**EvidenceQuality(StrEnum):**
Drei Werte: ESTIMATE / TESTED / PROVEN. Ursprünglich als `str, Enum` geschrieben
— ruff hat das mit UP042 korrekt geflgt. Fix: `StrEnum` aus Python 3.11+.

**UseCaseInput:**
- model_config: extra="forbid", str_strip_whitespace=True
- Pflichtfelder: title, submitter, department, current_state,
  desired_state, example_process
- Optionale Felder: time_savings_hours_per_case (ge=0.0),
  frequency_per_year (ge=0), evidence_quality, contains_pii
- max_length auf allen Textfeldern (200 für kurze, 2000 für lange)

### `tests/domain/test_use_case_input.py`
9 Tests in 3 Klassen, TDD-Ansatz (Tests vor Implementation):
- TestUseCaseInputValideEingaben: 2 Happy-Path-Tests
- TestUseCaseInputValidierung: 6 Fehler-Tests (leer, zu lang, extra-Feld,
  negativ)
- TestUseCaseInputModelValidate: 1 Test für model_validate() aus Dict

### Dependency hinzugefügt
`uv add pydantic` — war nicht installiert. uv hat pyproject.toml und
uv.lock aktualisiert.

## Was hat gut funktioniert?

TDD-Ansatz: Tests zuerst schreiben, dann sehen wie sie fehlschlagen
(ImportError), dann Implementation. Macht klar was das Schema können soll
bevor man es baut.

## Was war unerwartet?

Pydantic war nicht installiert — der erste pytest-Lauf hat mit
ModuleNotFoundError geendet. `uv add pydantic` hat das in Sekunden gefixt.
Merke: uv verwaltet Dependencies explizit, nichts kommt implizit mit.

ruff UP042: `str, Enum` ist technisch korrekt aber veraltet.
Python 3.11 hat StrEnum eingebaut — eine Klasse statt zwei als Basis.
